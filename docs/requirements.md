# Kansu: infinite scroll archiver 要件定義書

## 1. プロジェクトの背景と目的

多くのウェブサイト、特にSNSやECサイトでは、ユーザーがページをスクロールすると新しいコンテンツが動的に読み込まれる「無限スクロール」が採用されています。これによりユーザーはシームレスなブラウジング体験を得られますが、一方で、表示された情報を後からまとめて確認したり、整理したりすることが困難です。

本プロジェクト「Kansu (巻子)」は、この課題を解決するためのブラウザ拡張機能です。ユーザーが定義したルールに基づき、無限スクロールで表示されるウェブページから体系的にデータを抽出し、ローカル環境で効率的に管理（検索、ソート、エクスポート）する手段を提供します。これにより、ユーザーは大量の情報を手軽に収集・活用できるようになります。

## 2. ターゲットユーザー

特定のウェブサイト（例: SNS、ECサイト、ニュースサイトなど）から継続的に情報を収集し、後から参照・整理したいと考えている一般的なPCユーザーを想定しています。専門的な知識がなくても、直感的に利用できることを目指します。

## 3. 機能要件

### 3.1. データ抽出

- ユーザーが指定したウェブページ（URLパターンで指定）で機能を有効化できること。
- ユーザーが定義したルール（CSSセレクタ等）に基づき、ページ内の要素から特定のデータ（テキスト、リンク、画像URLなど）を自動で抽出できること。
- ページの初回読み込み時および、無限スクロールによって新しいコンテンツが読み込まれた際に、自動でデータ抽出処理が実行されること。

### 3.2. データ管理

- 抽出したデータをブラウザ内のデータベース（IndexedDB）に永続的に保存できること。
- 保存したデータに対して、キーワードによる全文検索ができること。
  - 検索対象の項目はユーザーが選択できること。
  - 日本語の表記揺れ（全角/半角、ひらがな/カタカナ）を吸収して検索できること。
  - 入力中に検索結果が更新されるインクリメンタル検索に対応すること。
- ユーザーが指定した項目を基準に、データの並び替え（昇順/降順）ができること。
- 大量のデータを効率的に表示するため、ページネーション機能を提供すること。
  - 1ページあたりの表示件数をユーザーが設定できること。

### 3.3. データと設定のインポート/エクスポート

- サービスごとに抽出したデータと、その抽出設定をファイルにエクスポートできること。
- エクスポートしたファイルをインポートし、データと設定を復元できること。

### 3.4. ユーザーインターフェース

- 拡張機能のメインUIの表示/非表示を切り替えられること。
- モダンで直感的なデザインのUIを提供すること。
- 操作の結果（成功、エラーなど）は、画面上に一時的に表示される通知（トースト通知）でフィードバックされること。

## 4. 非機能要件

### 4.1. パフォーマンス

- 大量のデータを扱っても、UIがスムーズに動作し、応答性が高いこと。
- データ抽出処理が、ウェブサイトの閲覧体験を著しく損なわないこと。

### 4.2. 互換性

- Google Chromeを主要なサポート対象とします。

### 4.3. セキュリティと倫理

- 抽出したデータは、ユーザーのローカル環境（ブラウザ内）にのみ保存し、外部のサーバーには一切送信しないこと。
- ウェブサイトのAPIを直接呼び出すなど、提供元が意図しない方法でのデータ取得は行わないこと。あくまでブラウザに表示された情報を対象とすること。

## 5. スコープ

### 5.1. スコープ内

- 上記の機能要件・非機能要件で定義されたすべての機能。

### 5.2. スコープ外（将来の検討事項）

以下の機能は本プロジェクトの初期スコープには含めず、将来的な拡張機能として検討します。

- データの一覧: 抽出したデータのDB一覧を表示する機能。
- ユーザーガイドの拡充: 汎用的な設定方法や、各ウェブサイトでの設定例を含む詳細なユーザーガイドの作成。
- スクロール自動化: 無限スクロールのスクロールを自動でおこなう機能。
- 国際化 (i18n): 複数の言語に対応するための国際化対応。
- クラウド同期: 設定や抽出したデータをクラウドストレージ（Google Drive, Dropboxなど）と同期する機能。
