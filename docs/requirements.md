# Kansu: infinite scroll archiver 要件定義書

## 1. 目的

本プロジェクト「Kansu (巻子)」は、無限スクロール型サイトで画面上に表示された情報を、ユーザー定義ルールに基づいて収集・保存・検索可能にするブラウザ拡張機能です。  
主目的は次の2点です。

- 後から再参照しづらい動的コンテンツを、ローカルに蓄積して再利用可能にする
- 非エンジニアでも扱えるUIで、抽出設定・検索・エクスポートまで完結できるようにする

## 2. 想定ユーザー

- SNS、EC、ニュースなどで継続的に情報収集するPCユーザー
- HTML/CSSセレクタに詳しくなくても、テンプレートベースで設定したいユーザー
- 収集データをローカル管理したいユーザー（外部サーバー送信なし）

## 3. 用語定義

- **サービス設定**: 特定サイト向けの抽出ルール群（URLパターン、セレクタ、主キー定義など）
- **抽出レコード**: 1つのコンテンツ要素から抽出された1件の保存データ
- **主キー**: 重複判定に使う一意キー（サービス設定で指定）
- **正規化文字列**: 検索向けに `NFKC` 正規化・表記揺れ吸収を施した文字列

## 4. 機能要件（MVP）

### 4.1. サービス設定管理

- **FR-01**: URLパターン単位でサービス設定を有効化/無効化できること
- **FR-02**: サービス設定として以下を保持できること
  - サービスID/名称
  - 対象URLパターン
  - 監視対象ルートセレクタ（MutationObserver対象）
  - アイテムセレクタ
  - フィールド定義（`text` / `linkUrl` / `imageUrl` / `regex`）
  - 主キーに使うフィールド名
- **FR-03**: 設定保存時に必須項目・セレクタ・主キー整合性を検証し、無効設定を保存しないこと

### 4.2. データ抽出

- **FR-10**: ページ初期表示時に抽出処理を実行できること
- **FR-11**: 無限スクロールによるDOM追加時に抽出処理を再実行できること
- **FR-12**: 同一主キーのレコードは重複作成せず、最新内容で更新（upsert）できること
- **FR-13**: 抽出処理はバッチ化し、短時間の大量DOM変更でも過剰に再実行しないこと

### 4.3. データ保存・検索・表示

- **FR-20**: 抽出レコードとサービス設定をIndexedDBに永続保存できること
- **FR-21**: キーワード検索を実行できること
  - 検索対象フィールドはユーザーが選択可能
  - 正規化文字列を用いて、全角/半角・ひらがな/カタカナ差を吸収
  - 入力中に結果が更新されるインクリメンタル検索
- **FR-22**: 昇順/降順のソートができること（文字列・数値混在を考慮）
- **FR-23**: ページネーションを提供し、1ページ件数をユーザーが選択できること

### 4.4. UI/操作

- **FR-30**: Popupから、現在タブのメインUI表示/非表示を切り替えできること
- **FR-31**: Optionsでサービス設定の追加/編集/削除ができること
- **FR-32**: 成功/失敗をトースト通知で即時フィードバックすること
- **FR-33**: キーボード操作で主要操作（検索、ソート、ページ移動）が実行できること

### 4.5. インポート/エクスポート

- **FR-40**: サービス単位で設定＋抽出データをJSONエクスポートできること
- **FR-41**: JSONインポート時に `schemaVersion` を検証すること
- **FR-42**: インポート時の重複主キーは「上書き（upsert）」で統一し、結果件数を通知すること

## 5. 非機能要件

### 5.1. パフォーマンス

- **NFR-01**: 1サービスあたり5,000件保存済みの状態で、検索結果更新が通常300ms以内（p95）であること
- **NFR-02**: ソート変更から再描画まで200ms以内（p95）を目標とすること
- **NFR-03**: MutationObserverコールバック内では重い処理を避け、抽出本体はバッチ処理へ遅延実行すること
- **NFR-04**: データ保存は可能な限り `bulkPut` + transaction でまとめ、書き込み回数を抑えること

### 5.2. 信頼性

- **NFR-10**: Manifest V3のService Worker停止/再開を前提に、重要状態を揮発メモリにのみ保持しないこと
- **NFR-11**: 主要メッセージ処理は失敗時に呼び出し元へエラー理由を返せること
- **NFR-12**: インポート処理はトランザクション単位で整合性を維持すること

### 5.3. セキュリティ・プライバシー

- **NFR-20**: 抽出データを外部サーバーへ送信しない（ローカル保存のみ）
- **NFR-21**: 権限は最小化し、不要な `host_permissions` を要求しないこと
- **NFR-22**: 表示済みDOMのみを対象にし、提供元が意図しないAPI直接アクセスは行わないこと
- **NFR-23**: インポートファイルはスキーマ検証し、異常値をそのまま反映しないこと

### 5.4. 互換性・運用

- **NFR-30**: Chrome（Manifest V3）を正式対象とすること
- **NFR-31**: CIで `check` / `lint-md` / `test` / `build` / `e2e` を通過できること

## 6. スコープ

### 6.1. スコープ内（MVP）

- サービス設定管理
- 無限スクロール抽出（手動スクロール前提）
- ローカル保存、検索、ソート、ページネーション
- サービス単位のインポート/エクスポート
- Popup/Options/ページ内メインUI

### 6.2. スコープ外（将来検討）

- スクロール自動化
- クラウド同期
- 複数ブラウザ対応（Firefox/Safari正式サポート）
- 国際化（i18n）
- 高度な分析機能（タグ付け、集計ダッシュボード）

## 7. 受け入れ基準（MVP）

- **AC-01**: 対応URLを開くと、設定したフィールドが自動抽出され、重複なく保存される
- **AC-02**: 既存5,000件データで、検索入力中に結果が実用的な速度で更新される
- **AC-03**: ひらがな/カタカナ、全角/半角差を含む語で検索しても期待結果に到達できる
- **AC-04**: PopupからUI表示を切り替えられ、Optionsで設定更新後に挙動へ反映される
- **AC-05**: エクスポートしたJSONを再インポートして、設定とデータが再現される
- **AC-06**: CIの必須ジョブが通ること
