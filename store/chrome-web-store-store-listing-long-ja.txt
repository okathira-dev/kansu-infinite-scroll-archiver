kansu-infinite-scroll-archiver

この拡張機能「Kansu（巻子）」は、無限スクロールのほか、一覧の追記や差し替えなどで自動的に更新される欄に表示された情報を、好きなルールで取り込み、ブラウザ内に蓄えてあとから検索・整理できる Chrome 向けの拡張機能です。
タイムラインや検索結果、商品一覧など、スクロールや自動読み込みなどで次々と表示が変わるページ向けに、ブックマークやスクリーンショットに頼らず「自分の端末の中にデータとして残す」ことを目指しています。後から参照しづらい動的な内容をローカルで再利用しやすくすること、そして検索やインポートエクスポートまで済ませられることを目的にしています。

想定している使い方:
タイムラインを読み進めながら、表示された投稿やカードのタイトル・本文・リンクなどを抜き出し、あとからキーワードで探し直す場面。ニュースで見出しや日付を蓄え、ブラウザを閉じたあとも一覧から検索する場面。調査や情報収集でサイトごとにルールを切り替え、設定とデータを JSON に書き出したり別の環境へ読み込んだりする場面。こうしたときに、サイト側にだけ頼らず自分の PC 内にデータベースとして持てるのが共通のメリットです。

インストールすると得られるメリット:
表示が自動で更新される一覧などを、閲覧しながら端末内へ蓄えられます。タブを閉じたあとも、拡張機能の画面からキーワードで探し直せます。セレクタに詳しくなくても、テンプレートに沿って「どのサイトのどの範囲を監視するか」「1 件の項目をどう表すか」「本文・リンク・画像・正規表現で何を保存するか」をオプションから定義できます。保存先はブラウザ内で、キーワード検索、昇順・降順の並べ替え、ページ送りで閲覧できます。検索は全角・半角やひらがな・カタカナのゆれをある程度吸収し、入力に合わせて結果が変わるリアルタイム検索に対応します。
設定と抽出データはサービス単位で JSON に書き出せ、同じ形式で別プロファイルや別デバイスに引き継げます。外部サーバーへデータを送ったり第三者と共有したりする機能はありません。

主な機能:
【サービス設定】URL のパターンごとに設定のオン・オフができます。名前、対象 URL、監視する範囲（ルート要素）、1 件分の項目の取り方、本文・リンク・画像・正規表現などのフィールド、重複を避けるための主キーなどを保存します。保存のときに必須項目や整合性を確認し、問題がある設定は保存されません。
【表示の更新に追従】最初の表示のあと、スクロールや自動読み込みなどで一覧に項目が足されたり内容が変わったりしたときも、ルールに従って取り込みを更新します。同じキーのデータは二重に作らず、新しい内容で上書きします。短時間に画面が大きく変わっても、まとめて処理するようになっています。
【検索・一覧・操作】検索に使うフィールドを選べます。文字列と数値が混在しても並べ替えでき、1 ページの件数も選べます。ポップアップから、今開いているタブ上にメイン画面の表示を切り替えられ、検索や並べ替え、ページ移動はキーボードでも操作できます。

はじめかた:
1. インストールしたら、拡張機能のオプションを開き、対象サイト用のサービス設定を追加します。
2. 設定した URL に合うページを開き、スクロールやタブ操作などで、取り込みたい項目が画面に現れるようにします。無限スクロールなど、表示が自動で更新される欄に出た項目がルールに従って保存されます（この拡張機能がスクロール操作を代わりに自動で行うことはありません）。
3. ポップアップからメイン画面を表示し、蓄えたデータを検索・並べ替えして使います。
4. 必要なら JSON の書き出しや読み込みでバックアップや移行を行います。

プライバシーとデータについて:
画面上に表示されている内容だけを、あなたのルールに従って読み取り、端末内の IndexedDB に保存します。サイトが公開していない API を裏から呼び出すことはしません。外部サーバーへ収集データを送ることはなく、設定・抽出・検索・インポートやエクスポートは端末内で完結します。
